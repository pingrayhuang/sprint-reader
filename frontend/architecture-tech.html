Sprint Reader 系統架構（技術版）

單體 FastAPI + SQLite。下方三張圖：① 完整 API 呼叫序列（swim lane）、② DB schema、③ Sprint 狀態機。 每個欄位 / endpoint / 表名都與原始碼完全一致。

① 完整 API 串接序列（單次 Sprint + Quiz）
每一列 = 一個使用者動作觸發的完整 request/response + DB 寫入。

STEP 1index.html 載入
顯示 TOC + ring + 今日推薦
GET/api/module→ modules[]: id, title, duration_sec, page_count, quiz_count GET/api/stats/progress GET/api/stats/chapters GET/api/stats/recommend
（純查詢，無狀態變更）
SELECT × MicroModules、SprintSessions、LearningJourney_Map
STEP 2splash.html → 點「開始」
POST/api/sprint/start
body: { agent_id, module_id } → sprint_id (uuid4)
session_manager.start_sprint()
生 uuid、tab_switch_count=0、status=reading
INSERT SprintSessions
(sprint_id, agent_id, module_id, start_timestamp, completion_status='reading')
STEP 3reader.html：取本章卡片渲染
GET/api/module/{module_id}→ pages[]: sequence_number, title, body, highlight
-
SELECT × FlashcardPages WHERE module_id=?
STEP 4使用者切到 Line（Visibility API）
document.hidden = true
POST/api/sprint/telemetry
body: { sprint_id, event: "tab_switch" } → { tab_switch_count }
session_manager.record_tab_switch()
原子遞增（DB-level UPDATE ... +1）
UPDATE SprintSessions SET tab_switch_count = tab_switch_count + 1
STEP 5點「✅ 進入測驗」或倒數完
（目前倒數 0 不自動觸發，由使用者主動點）
POST/api/sprint/complete
body: { sprint_id, status: "finished_early" }
POST/api/handoff/to-quiz
→ { quiz_session_id }
complete_sprint()：設 end_timestamp
create_quiz_handoff()：生 quiz_session_id
UPDATE SprintSessions SET end_timestamp, completion_status='handoff'
INSERT LearningJourney_Map(sprint_id, quiz_session_id)
STEP 6handoff.html → quiz.html
GET/api/quiz/{module_id}→ questions[]（不含 correct_index）
-
SELECT × QuizQuestions WHERE module_id=? ORDER BY sequence_number
STEP 7使用者全部答完 → 按提交
POST/api/quiz/submit-batch
body: { quiz_session_id, answers: [{question_id, chosen_index}] }
POST/api/quiz/finalize
→ score, total, responses[] (含 correct_index + explanation)
judge each: chosen == correct
total = len(responses)（bug fix 0415）
INSERT × QuizResponses
UPDATE LearningJourney_Map SET score, total_questions
STEP 8result.html 顯示答對率 + 逐題檢討
結果直接由 finalize 回傳，無額外 API
-
（僅讀 sessionStorage.result）
ⓘ 關鍵 race-safe 設計：STEP 4 的 tab_switch_count 用 DB-level UPDATE 遞增，不是先 SELECT 再 UPDATE，避免使用者快速切換造成漏計。 STEP 5 的 end_timestamp 由伺服器蓋章，前端改 JS 時鐘無效。
② 資料層 · 6 張 SQLite 表
init_db.py 建表 + seed。中央表 LearningJourney_Map 是 P2/P3 橋接點。

MicroModules

  • module_id (PK)
  • title
  • description
  • duration_sec · page_count · quiz_count
  • source_doc · domain_tags · sort_order
6 章 metadata

FlashcardPages

  • page_id (PK)
  • module_id (FK)
  • sequence_number · title · body · highlight
30 張卡（5×6 章）

QuizQuestions

  • question_id (PK)
  • module_id (FK)
  • sequence_number · stem · options_json
  • correct_index · explanation · source_page_seq
每章 ≥ 3 題，真題庫

SprintSessions ★

  • sprint_id (PK, uuid)
  • agent_id · module_id
  • start_timestamp · end_timestamp
  • tab_switch_count
  • completion_status
本專案核心：一次閱讀 = 一筆

QuizResponses

  • response_id (PK)
  • quiz_session_id · question_id
  • chosen_index · is_correct · answered_at
每題一筆，錯題分析用

LearningJourney_Map ★

  • journey_id (PK)
  • sprint_id · quiz_session_id
  • score · total_questions
  • created_at
橋接表：P3 閱讀 ⟷ P2 測驗 ⟷ 分數。未來 AI 餵料源。
