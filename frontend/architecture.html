5 個關鍵設計決策（面試可深入）

① 後端蓋章 timestamp（防作弊）
Start / end 都由 session_manager 寫入。前端改 JS / 時鐘也改不到真實閱讀時間。
② Visibility API + 即時累計
切走就 POST 一次 tab_switch，切回就恢復計時。分心次數 = 行為遙測資產。
③ LearningJourney_Map（P2/P3 橋接）
一張表同時存 sprint_id + quiz_session_id + score，未來串 AI 做個人化推薦。
④ 單體 + SQLite（面試情境最優解）
零外部依賴、一鍵 bash run.sh 可 demo。正式版再切 Postgres + Redis。
⑤ 題目來自真題庫
「業務員考試考古題」直接 seed 進 DB，被問「題目哪來的」有可信答案。
