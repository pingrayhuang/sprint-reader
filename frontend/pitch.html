03 · 核心矛盾
傳統訓練量「出席率」，我們量「記憶效果」
這不只是功能差異，是衡量標準的根本轉移 — 改變了整個訓練 ROI 的計算方式
傳統 e-learning 現狀
  • 出席率 100% = 目標達成，點完課程即算合規
  • 沒有即時測驗，閱讀等於瀏覽，無法確認是否理解
  • 所有人看同一套內容，個人弱點被平均值掩蓋
  • 72 小時後知識消退，沒有任何補強或提醒機制
  • 法遵稽核只能查「有無點開」，無法驗證真實吸收
Sprint Reader 新邏輯
  • 閱讀後立即接 3 題測驗，強制主動記憶提取
  • 每題作答全記錄進 DB，合規稽核具備可信憑據
  • 個人答錯率建立弱點圖譜，差異化複習
  • 間隔重複推薦，在遺忘曲線觸底前主動介入
  • 分心次數偵測行為異常，早期識別高風險學習者
MA 洞察：凱基金合規訓練的問題，不在預算，在衡量指標。完訓率量的是「行為」，答對率量的是「效果」。把指標從前者換成後者，才能讓 CEO 看到真正的訓練 ROI，也才能讓法遵部門的工作從成本中心，變成可量化的風險管理資產。

04 · 解法

⏱

7 分鐘限時衝刺

110 頁考照筆記切成 6 章 × 5 張卡，每章 7 分鐘。「認知負荷理論」指出工作記憶容量有限，7 分鐘恰好是一個不超載的學習單元，填滿通勤空檔。

↗ 完訓率 × 降低阻力
🎯

即時測驗 — Active Recall

讀完立即接 3 題 quiz。「主動提取效應」顯示：考試本身就是最有效的記憶鞏固方式。搭配詳細解釋與原文卡片回溯，讓錯誤成為學習機會。

↗ 記憶留存率 3×
🔁

間隔重複推薦

根據「答錯率 × log（距上次複習天數 + 1）」自動推薦章節。系統決定「今天讀哪章」，不靠意志力，在遺忘曲線觸底前主動介入。

↗ 長期留存率

07 · 技術亮點 01 / 03
一個 UX 設計，蘊含三層商業意義
Browser Visibility API — 把「業務員切到 Line」這個動作，轉化成可分析、可預警的學習行為數據
怎麼做到的
監聽 document.visibilityState，切換分頁或 app 時計時器自動暫停；切回後接續，體驗流暢不中斷。每次切換事件 POST 到後端 tab_switch_count++。
為什麼這個數據有商業價值
第一層：UX — 暫停計時保護使用者體驗，不懲罰被打斷的業務員。
第二層：行為分析 — 分心次數高的學習者，測驗答錯率顯著偏高，可作為「需個別輔導」的早期預警訊號。
第三層：AI 特徵 — tab_switch_count 成為 RL 推薦引擎的輸入特徵，讓系統識別「高風險」學習模式。

08 · 技術亮點 02 / 03
計時器由後端蓋章，前端改 JS 也沒用
Server-Authoritative Timing — 保障閱讀紀錄真實性，讓學習完訓具備法遵稽核效力
設計邏輯
所有時間戳由後端 session_manager.py 記錄與驗證。即使使用者修改瀏覽器時間、或用開發者工具竄改 JS，後端的閱讀時長仍正確。前端只負責顯示倒數畫面，不握有任何可信時間資訊。
三表分離 = 三個部門使用者
這個 schema 設計不是偶然 — 我在設計時刻意想到三個下游使用者：
SprintSessions → 法遵部：完訓稽核憑據，時間戳可信
QuizResponses → 資料部：個人答題序列，餵 AI 推薦
LearningJourney_Map → 教育訓練部：閱讀與測驗橋接，追蹤學習旅程
這是 MA 跨部門思維直接體現在 schema 設計上。

09 · 技術亮點 03 / 03
系統自動決定「今天要讀哪章」
間隔重複推薦演算法 — 把 Ebbinghaus 學習科學，落實為工程實作
運作邏輯
每次完成 quiz 後，後端更新各章節掌握度，根據兩個維度計算出推薦分數，自動在首頁顯示「今日推薦」章節，讓業務員不需要自己決定讀什麼，系統在遺忘發生前介入。
推薦優先度公式
Priority = 答錯率 × log（距上次天數 + 1）
答錯率 ↑
這章還不熟
需要優先複習
天數 ↑
越久沒讀
越快被推薦
Phase 3 升級路徑
當前為規則式算法（可解釋性高、易 debug）。收集足夠行為序列後，可升級為強化學習（RL），動態預測每位員工的遺忘時點。

Phase 1 · MVP — 學習閉環已驗證

7 分鐘滑卡閱讀 + 3 題即時測驗 + 逐題解釋 + 規則式間隔重複推薦 + 完整行為遙測（分心次數、閱讀時間、完成狀態）。End-to-end 可 demo，DB 紀錄可稽核。

Phase 2 · 內容自動化 — 降低 80% 人工編輯成本

接入凱基真實合規文件，用 LLM（Content Agent + Quiz Generator）自動把長文切成 7 分鐘 sprint 包。教育訓練部門從「手工編輯 5 天」到「上傳 PDF，1 小時產出課程」。

6 個月

Phase 3 · RL 推薦引擎 — 個人化學習體驗

把規則式算法升級為強化學習，動態預測每位員工「即將遺忘」的精確時點，在最佳時機主動推播通知，不再依賴業務員自己記得打開 App。

12 個月

Phase 4 · 風險預警 — 合規訓練成為風險管理資產

學習行為訊號接入 Risk Scorer，提早預警違規高風險個案，建立個人化推薦引擎。合規訓練從被動的「每年一次義務課程」，升級為主動的「持續風險監控系統」。
